MongoDB中的Shell

可用来预MongoDB实例交互。

运行shell

运行mongo启动shell，启动时必须保证mongod是启动的。
Shell是一个功能完备的javaScript解释器，可以运行任意的js程序。

MongoDB客户端

查看db当前指向哪个数据库，使用db命令
选择数据库：use foobar

创建

Insert函数将一个文档插入到集合中。

读取

Find和findOne方法可以用于查询集合里的文档，若查一个用findOne，使用find时最多20个匹配文档

更新

Update方法接收至少两个参数：第一个时限定条件，第二个是新的 文档。

删除

Remove方法将文档从数据库中永久删除，可接受一个参数作为限定条件

数据类型

基本数据类型

  1. null：表空值或不存在的字段
  2. 布尔：true | false
  3. 数字：默认使用64位浮点型，可使用NumberInt类（4字节的整数）或NumberLong类（8字符带符号整数）
  4. 字符串
  5. 数组：数据的列表或数据集
  6. 对象

    日期

    new Date()创建日期对象（调用constructor），而Date()返回的是日期字符串（没调用constructor），这是由js的工作机制决定的

    数组

    是一组值，可以作为有序对象，也可作为无序对象，数组中可以包含不同类型的元素：{"things": ["pie", 3.14]}MongoDB可以深入数组内部对其内容进行操作

    内嵌文档

    文档可以作为键的值，这种的文档就是内嵌文档，可以使数据组织方式更加自然。MongoDB也能够理解内嵌文档的结构，并深入其中构建索引、执行查询等
    优点：使用得当的话，内嵌文档会使信息的表示更加自然，也可能会提高效率
    缺点：修改信息不方便

_id

必须的，任意类型，默认是ObjectId对象，在每个集合中具有唯一性

ObjectId

不同的机器都用全局唯一的同种方法方便地生成它。MongoDB不采用id自增是因为在多个服务器上同步自增加主键值既费力又费时，所以MongoDB的初衷就是用作分布式数据库，所以能够在分片的环境中生存唯一的标示符非常重要。ObjectId使用12字节，又一个24个16进制数字组成的字符串（每个字节可以存储2个16进制数字）。ObjectId的12字节生产方式是：

  • 时间戳：从标准纪元开始，秒为单位，与随后的5字节，提供了秒级别的唯一性。大概以时间先后顺序插入的。包含了文档创建的时间。
  • 机器：所在主机的唯一标识符。
  • pid：ObjectId的进程的进程标识符
  • 计数器：保证相同进程同一秒产生的ObjectId也是唯一的，一秒钟最多允许每个进程拥有2563个不同的ObjectId。

自动生成_id

如果没有“_id”，系统会自动帮你创建一个。可以由服务器来做，但通常会在客户端由驱动程序完成。这一做法就体现了MongoDB的哲学：能交给客户端驱动程序来做的事情就不要交给服务器来做。
